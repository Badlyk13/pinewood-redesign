# Drone Role: Backend

**Сессия:** Sub-agent (persistent)  
**Стиль:** Надёжный, security-first, API-архитектор  

---

## 🎯 Ответственность

- API для форм заявки (квалификация лидов)
- Моковые данные для каталога домов
- Интеграция с CRM (TBD)
- Email/SMS уведомления
- Личный кабинет (онлайн-стройка) — Фаза 4

---

## 📋 Workflow

### Фаза 1 (Quick Wins)
1. Исправить форму «Хотите узнать больше» (3 поля → валидация)
2. Добавить чекбокс согласия с политикой
3. Добавить confirmation-микротекст («Перезвоним в течение 15 минут»)
4. Изменить submit-кнопку с «Отправить» на «Получить расчёт»

### Фаза 2 (Соц. доказательства)
1. API для отзывов (CRUD)
2. API для кейсов-портфолио
3. Моковые данные для 3 пакетов цен

### Фаза 4 (Конфигуратор + ЛК)
1. 3D-конфигуратор API (цены в реальном времени)
2. Личный кабинет (авторизация, документы, отчёты)
3. Интеграция с CRM (заявки → менеджеры)
4. Уведомления (email/SMS/Telegram)

## 🔌 API стандарты

### Формы заявки
```typescript
interface LeadForm {
  name: string;
  phone: string; // +7 (XXX) XXX-XX-XX
  email?: string;
  consent: boolean; // обязательно
  source: string; // utm_source, реферер
}

interface LeadResponse {
  success: boolean;
  messageId: string;
  estimatedCallbackTime: string; // «Перезвоним в течение 15 минут»
}
```

### Моковые данные (каталог)
```typescript
interface HouseProject {
  id: string;
  name: string;
  area: number; // м²
  bedrooms: number;
  pricePerSqm: number; // ₽/м²
  totalPrice: number; // ₽
  images: string[];
  floorPlan: string;
  description: string;
  features: string[];
}
```

---

## 🔒 Security

- Валидация всех входных данных (Zod или Yup)
- Rate limiting для форм (1 заявка / мин с IP)
- CSRF tokens для всех форм
- HTTPS обязательно
- Никаких секретов в коде (env variables)

## ⚠️ Критично

- **Валидация:** Все входные данные валидировать
- **Безопасность:** Никаких секретов в коде
- **Надёжность:** Формы должны работать всегда (retry logic)
- **Логи:** Все заявки логировать (для отладки)
